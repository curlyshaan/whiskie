## 🔄 Decision Flow (How It Works)

### Before Analysis
1. Load portfolio from database
2. Load watchlist from database
3. Check if any watchlist stocks hit target prices

### Phase 1: Sub-Industry Screening
1. Fetch market indices + portfolio stock prices
2. Fetch market news
3. Show Opus:
   - Current portfolio
   - Watchlist with target prices
   - 40 available sub-industries
   - Market context
4. Opus identifies:
   - 3-5 promising sub-industries
   - 15-20 specific stocks to analyze
5. Fetch real-time prices for identified stocks

### Phase 2: Deep Analysis & Decisions
1. Opus analyzes with real-time prices
2. Opus decides for each stock:
   - **BUY NOW**: Execute trade immediately
   - **ADD TO WATCHLIST**: Monitor for future opportunity
   - **IGNORE**: Not interesting
3. Parse and execute:
   - Trade recommendations → Execute → Send email
   - Watchlist updates → Save to database
4. Log everything with token usage

### Next Analysis
- Watchlist is reviewed first
- Stocks at target prices get priority
- Watchlist grows/shrinks based on opportunities

## ⚠️ What Could Go Wrong (& Solutions)

### Issue: Opus doesn't use watchlist format

**Solution**: Make format more prominent or add validation
**Impact**: Watchlist won't populate, but trades still work

### Issue: Too many stocks identified
**Likelihood**: Low (prompt limits to 15-20)
**Solution**: Code takes first 20 if exceeded
**Impact**: None (handled automatically)

### Issue: Watchlist grows too large
**Likelihood**: Medium (over time)

**Impact**: Minor (just more items to review)

### Issue: Token costs increase

**Solution**: Monitor usage in database, adjust if needed
**Impact**: Trackable via new token columns
