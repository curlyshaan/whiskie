# Whiskie Bot - Deployment Summary & Design Review

## Overview
This deployment adds **sub-industry classification** and **watchlist functionality** to enable extensive stock screening (100-200+ stocks) while maintaining a focused portfolio of 10-12 positions.

## 🗄️ Database Changes

### 1. `ai_decisions` Table - New Columns
```sql
ALTER TABLE ai_decisions
ADD COLUMN input_tokens INTEGER,
ADD COLUMN output_tokens INTEGER,
ADD COLUMN total_tokens INTEGER,
ADD COLUMN cost_estimate DECIMAL(10, 4),
ADD COLUMN duration_seconds INTEGER;
```

**Purpose**: Track AI usage and costs for each analysis

### 2. `watchlist` Table - New Table
```sql
CREATE TABLE watchlist (
  id SERIAL PRIMARY KEY,
  symbol VARCHAR(10) UNIQUE NOT NULL,
  sub_industry VARCHAR(100),
  current_price DECIMAL(10, 2),
  target_entry_price DECIMAL(10, 2),
  target_exit_price DECIMAL(10, 2),
  why_watching TEXT,
  why_not_buying_now TEXT,
  status VARCHAR(20) DEFAULT 'watching',
  added_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  last_reviewed TIMESTAMP,
  price_when_added DECIMAL(10, 2),
  highest_price DECIMAL(10, 2),
  lowest_price DECIMAL(10, 2)
);
```

**Purpose**: Track stocks Whiskie is monitoring but not ready to buy yet

**Example Use Case**:
- AAPL at $280, target entry $250, target exit $320
- Why watching: "Strong fundamentals, iPhone 16 cycle"
- Why not buying now: "Waiting for pullback to $250 support level"

## 🔄 Analysis Flow (Complete)

### Phase 1: Sub-Industry Screening
```
1. Load portfolio from database
2. Fetch watchlist from database
3. Fetch market indices (SPY, QQQ, DIA, IWM, VIX, TLT, GLD, USO)
4. Fetch portfolio stock prices
5. Refresh portfolio prices with live data
6. Fetch market news (Tavily API)
7. Build Phase 1 prompt with:
   - Current portfolio status
   - Watchlist with target prices
   - Market context (indices)
   - Recent news
   - All 40 sub-industries available
8. Send to Claude Opus with extended thinking
9. Opus identifies:
   - 3-5 promising sub-industries
   - 15-20 specific stocks to analyze
10. Extract tickers from Opus response
```

### Phase 2: Deep Analysis & Trade Decisions
```
1. Fetch real-time prices for all identified stocks
2. Build Phase 2 prompt with:
   - Full market data (indices + portfolio + identified stocks)
   - Watchlist context
   - Investment rules and guidelines
3. Send to Claude Opus with extended thinking (50k tokens)
4. Opus provides:
   - Watchlist updates (stocks to monitor)
   - BUY recommendations (stocks to buy NOW)
   - SELL/TRIM recommendations
   - Sector analysis
5. Parse watchlist items → Save to database
6. Parse trade recommendations → Execute trades → Send emails
7. Log analysis with token usage to database
```

## ⚠️ Potential Issues & Solutions

### Issue: Opus doesn't use watchlist format
**Solution**: The prompt clearly shows the format with example. If Opus ignores it, we can make the format more prominent or add validation.

### Issue: Too many stocks identified in Phase 1
**Solution**: Prompt limits to 15-20 stocks. If Opus exceeds, we take first 20.

### Issue: Watchlist grows too large
**Solution**: Add periodic cleanup (remove stocks that haven't been reviewed in 30 days).

### Issue: Token costs too high
**Solution**: Monitor token usage in database, adjust thinking budget if needed.
